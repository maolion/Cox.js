**Cox 框架设计的原则**
-   模块化
-   风格统一
-   不求精简只求实用
-   不污染环境提供的内置类型
-   不滥用全局标识
-   严密测试功能模块
-   框架的功能体系应该是足够广度与深度满足实际使用需求
-   跨环境(框架提供的全部模块并非都具有跨环境的兼容支持)
